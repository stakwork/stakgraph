### how to herd your agent flock

By the end of this year, agents will outnumber humans in the AI-pilled enterprise. Autonomous agent swarms happened so fast, many organizations haven't quite figured out how to manage this new superpower. Letting agents run free in the cloud enables massive parallel experimentation, and vastly accelerated product development. In our Hive system, we instruct agents to go research our current stack, pull in dependecies, and come back with fully built prototypes of features we are working on. This leaves more room for us humans to do what we are best at: playing with the product, communicating with real users, and understanding deeply WHY we are building it in the first place.

However, agents roaming free in the cloud can also lead to catastrophic side effects or runaway costs. When agents ran only on laptops, there were naturally scoped to the humans who ran them. But autonomous cloud agents, given bigger and more sophisticated toolboxes, are becoming more likely to escape the bounds of what they were intended to do. We solve this through a tiered authorization framework that puts humans at the center, not LLMs.

### architecture of layered autonomy

Agentic work in Hive is completed within the context of a Workspace (think a "team" or "product"), each which runs its own LLM gateway, through which all tokens flow. Whether an agent is using Claude, GPT, Gemini, or any other model, it derives its permissions and capabilities from a specific human in the hive ecosystem. Every single agent action or sub-agent workflow can be tied back to a human who authorized it in the first place! The most elegant solution here is a macaroon token with nested signatures: at the top level, an organization signs off on the work that a human is allowed to do. The human then signs off on the capabilities of an agent. The agent itself can attenuate these capabilities for a sub-agent (unable to broaden the scope of what it is allowed to do). In this way, every action taken by an agent is cryptographically traceable back to the individual contributor in an organzation (or across a few organizations using multisig!). The token gateway ensures that sessions can be killed or permissions revoked instantly.

Agent budgets are also cryptographically stamped into the macaroon itself, so there is no possible way an agent can outlive the scope of what it was assigned to do. Every file read, knowledge base query, git commit, sub-agent invocation, etc can be logged, hashed, and linked directly back to the organization, human, permission set, and session in which it lives. An agent registry per workspace allows organization leaders to explicitly define the capabilities that each agent and human in the system has.

This is how we keep our ever-growing flock of agents in line. It allows us to work in a completely flat team structure, where access is delegated up-front, and everyone knows exactly what they are able to do.
